## Minimum Prototype



### 1. Introduction

In recommender systems like twitter or reddit, the rewarding mechanism creates a competitive environment among creators which affect their production choices and, consequently, content distribution. To study on this phenominon, we plan to use Large Language Models to **simulate the users and content creators** in this kind of competition. 

The overall system can be devided into 3 parts: **Users**, **Content Creators**, **Rewarding Mechanism**.(In this minimum prototype, we **do not consider any recommending algorithms**, i.e. every user is accessible to all the posts generated by all the content creators.)

### 2. LLMs As Users

#### (0) Overview: what the user agents need to do?

+ Firstly, users are exposed only to the observable features of each post (e.g., \textbf{the title}) and make a \texttt{click} decision based on these cues. 
+ Then, the \textbf{abstract} is revealed only after a click occurs, at which point the user can choose whether to \texttt{like} the post.



#### (1) User profile Initialization

Using the reddit dataset "Reddit Corpus (small)" (from <url>https://convokit.cornell.edu/documentation/reddit-small.html), we initialize the user profile and make sure the user agents can act(click and view, upvote) just like the real users. 

##### a. data preparation

By the functions provided by Convokit, Reddit Corpus(small)(from <url>https://convokit.cornell.edu/documentation/reddit-small.html), we can easily aggregate the posts and comments of each user. Furthermore, we can know the parent text(the text that the comment is replying to). 

So for each user, we can get:
1.user_name: `string`
2.posts: `list` of all the posts the user had created
3.comments: `list` of all the (comment_text, parent_text) pairs. The comment_text and parent_text are both strings.

**What you need to do in this step: write python scripts to prepare the data, which should be stored in a proper file format.**

##### b.persona generation

Next, LLMs are used to generate user profiles. In the file `generate_persona.yml`, I have provided you a comprehensive prompt to request LLMs to do this job. However, I haven't clearly explained in the prompt the strucure of the data. So I'm worried that the LLM may not know the meaning of parent text.

**What you need to do in this step: 1.modify the `generate_persona.yml` file. 2.write python scripts to request the LLM to generate each user's profile**.

##### c. Synthesizing click & like data

Due to the fact that the dataset lacks the information needed to track which user clicked & liked whose post, we need to synthesize some click & like data, which can explicitly guide the user agent to click & like in a consistant manner.

For each user (we have got his profile in step b):

+ randomly select a certain number of comments from the dataset;
+ Feed this user's profile and these comments into the LLM;
+ Require the LLM to generate: whether to click; If so, whether to give a like. 
+ For each user, LLM should return JSON: {"text": string, "click": 0, "reason": string} or {"text": string, "click": 1, "like": binary, "reason": string}
+ Store all the users' click & like data into a file.

**What you need to do in this step: 1. design the prompt (in yml file) to require LLMs to generate synthetic click & like data; 2.write python scripts to requests the LLMs to generate synthetic click & like data**

### 5. Simulation Procedure

```latex
\item \noindent\textbf{Notation.} \\
		Let \textbf{$R$} be the total number of rounds and $n$ be the number of content creators. Let $\mathbf{P}^r$ denote the set of posts created in round $r$ (with $\mathbf{P}^0$ the initialization). For each creator $i$, let $\mathbf{P}^r_i$ denote creator $i$'s post in round $r$. Let $\mathcal{F}^r_i$ denote the set of users who viewed and scored creator $i$'s post in round $r$. Let $\sigma^r_i(x)$ denote the rate user $x$ gives content creator $i$ in the $r$th round(like for 1, otherwise 0). Let $M$ be a rewarding mechanism, and let $M^r_i$ be creator $i$'s reward in round $r$ computed by
\[
M^r_i \;=\; \mathbf{E}_{x \in \mathcal{F}^r_i}\!\bigl[M(\sigma^r_i, \sigma^r_{-i})\bigr], \quad i=1,\ldots,n .
\]
```



```latex
\caption{Simulation Procedure}
                        \label{alg:fp}
                        \begin{algorithmic}[1]
                        \State generate initial posts $\mathbf{P}^0$ (without any prior knowledge regarding users).
                        \For{$r = 1$ to $R$}
                          \State Users observe posts $\mathbf{P}^{r-1}$ and act (click & like) on each post, inducing $\{\mathcal{F}^r_i\}_{i=1}^n, \{\sigma^r_i\}_{i=1}^n$.
                          \State Compute rewards $\{M^r_i\}_{i=1}^n$ according to the rewarding mechanism $M$.
                          \State Each content creator $i$ generates a new post $\mathbf{P}^r_i$ as a best-response to the history $\{(\mathbf{P}^k_i, M_i^k)\}_{k=0}^{r-1}$.
                        \EndFor
                        \end{algorithmic}
                    \end{algorithm}
```
